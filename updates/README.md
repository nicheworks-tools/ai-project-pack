# Updates

`updates/` is an append-only log of meaningful safe-update and handoff passes.

## Rules
- Add one new dated markdown file per meaningful pass.
- Do not rewrite old update files except to correct factual errors.
- Include: summary, touched files, safe rationale, unresolved items, and next action.
- Reference supporting sources when claims depend on evidence.

## Naming
Use `YYYY-MM-DD-short-label.md` (example: `2026-04-07-first-safe-update.md`).

## Template
Start from `templates/update-entry.md`.
