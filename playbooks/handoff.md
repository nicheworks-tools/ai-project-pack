# Handoff Playbook

## Use when
Another model or person needs to restart work quickly and safely.

## Read first
`../current-truth.md`, `../pending.md`, `../next-actions.md`, latest relevant file in `../updates/`

## A good handoff
- current truth is short and current
- unresolved items are still visible
- next actions can be started without reinterpretation
- latest update log explains the last meaningful pass

## A bad handoff
- current truth contains history and wishlists
- pending items silently disappeared
- next actions are vague
- no update log explains recent edits
