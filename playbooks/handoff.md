# Handoff Playbook

## When to use
Use when switching models, maintainers, or sessions where restart speed matters.

## What to read first
- `current-truth.md`
- `decisions.md`
- `pending.md`
- `next-actions.md`
- latest relevant file under `updates/`

## Likely files to update
- `next-actions.md` (to remove ambiguity)
- `updates/<date>-handoff.md`

## Good mini-example
"Append a handoff update with three sections: stable now, unresolved now, first action next session."

## Bad mini-example / failure mode
"Handoff in chat only and skip writing an update log" (context is lost at restart).
